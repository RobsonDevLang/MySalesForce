# RF12: Listagem de Produtos Ativos Disponíveis

**Tipo:** Feature  
**Camada:** Backend

---

## Objetivo

Implementar uma funcionalidade no backend responsável por disponibilizar a listagem de produtos ativos cadastrados no sistema.

---

## Descrição

A funcionalidade permitirá que clientes da API consultem todos os produtos que estejam com status ativo, retornando apenas os registros disponíveis para utilização no sistema.

---

## Requisitos Funcionais

### RF12.1 - Consultar Produtos Ativos

O sistema deve disponibilizar um endpoint para retornar a lista de produtos ativos.

### RF12.2 - Retornar Informações do Produto

A listagem deve conter os seguintes dados:

- Identificador
- Nome
- Descrição
- Preço atual
- Desconto
- Categoria
- Status
- Lista de imagens (quando existir)

### RF12.3 - Filtrar Produtos Inativos

O sistema não deve retornar produtos marcados como inativos.

---

## Regras de Negócio

### RN12.1

Somente produtos com status **ATIVO** poderão ser retornados pela consulta.

### RN12.2

Produtos excluídos logicamente não devem aparecer na listagem.

### RN12.3

Caso não existam produtos ativos cadastrados, o sistema deverá retornar uma lista vazia.

---

## Critérios de Aceitação

### Cenário 1: Existem Produtos Ativos

**Dado** que existam produtos ativos cadastrados

**Quando** a consulta for realizada

**Então** o sistema deverá retornar a lista de produtos ativos.

---

### Cenário 2: Existem Produtos Ativos e Inativos

**Dado** que existam produtos ativos e inativos cadastrados

**Quando** a consulta for realizada

**Então** o sistema deverá retornar apenas os produtos ativos.

---

### Cenário 3: Não Existem Produtos Ativos

**Dado** que não existam produtos ativos cadastrados

**Quando** a consulta for realizada

**Então** o sistema deverá retornar uma lista vazia.

---

## Regras de Retorno da API

### Sucesso

#### 200 - Produtos Encontrados

Retorna uma lista contendo os produtos ativos disponíveis.

#### 200 - Nenhum Produto Encontrado

Retorna uma lista vazia.

```json
[]
```

### Erro

#### 500 - Erro Interno do Servidor

Retornado quando ocorrer um erro inesperado durante o processamento da requisição.

---

## Impacto Técnico

### Backend

Implementação de:

- Endpoint de consulta de produtos
- Camada Service
- Camada Repository
- Filtro por status ativo
- DTO de resposta

---

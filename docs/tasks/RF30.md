# RF30: Listagem de Produtos Ativos Disponíveis

**Tipo:** Feature
**Camada:** Frontend

---

## Objetivo

Implementar a interface responsável por consumir a listagem de produtos ativos disponibilizada pela API e exibir os produtos disponíveis para o usuário.

---

## Descrição

A funcionalidade permitirá que o usuário visualize todos os produtos ativos cadastrados no sistema através da interface da aplicação.

Os dados serão obtidos por meio da API de produtos e apresentados em uma listagem contendo as informações disponíveis para consulta.

---

## Requisitos Funcionais

### RF30.1 - Exibir Produtos Ativos

O sistema deve exibir na interface a lista de produtos ativos retornados pela API.

### RF30.2 - Exibir Informações do Produto

A listagem deve apresentar os seguintes dados:

* Nome
* Descrição
* Preço atual
* Desconto
* Categoria
* Status
* Lista de imagens (quando existir, quando não existir Ocultar o campo)

### RF30.3 - Não Exibir Produtos Inativos

Produtos inativos não devem ser exibidos na interface.

---

## Regras de Negócio

### RN30.1

Somente produtos retornados pela API com status **ATIVO** poderão ser exibidos.

---

## Critérios de Aceitação

### Cenário 1: Existem Produtos Ativos

**Dado** que existam produtos ativos cadastrados

**Quando** a tela for carregada

**Então** o sistema deverá exibir a lista de produtos ativos.

---

### Cenário 2: Existem Produtos Ativos e Inativos

**Dado** que existam produtos ativos e inativos cadastrados

**Quando** a tela for carregada

**Então** o sistema deverá exibir apenas os produtos ativos retornados pela API.

---

### Cenário 3: Não Existem Produtos Ativos

**Dado** que não existam produtos ativos cadastrados

**Quando** a tela for carregada

**Então** o sistema deverá exibir uma lista vazia ou mensagem informando que não existem produtos disponíveis.

---

## Integração com API

### Endpoint Consumido

```http
GET /product/active
```

### Sucesso

#### 200 - Produtos Encontrados

A interface deverá exibir a lista de produtos retornada pela API.

#### 200 - Nenhum Produto Encontrado

A interface deverá exibir uma lista vazia ou mensagem informativa.

```json
[]
```

### Erro

#### 500 - Erro Interno do Servidor

A interface deverá exibir uma mensagem de erro apropriada ao usuário.

---

## Impacto Técnico

### Frontend

Implementação de:

* Serviço de consumo da API de produtos
* Tela/componente de listagem de produtos
* Mapeamento do DTO de resposta
* Tratamento de carregamento (loading)
* Tratamento de erros da requisição
* Exibição dos produtos retornados
* Exibição de estado vazio quando não houver produtos
